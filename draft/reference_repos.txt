1. SkillRL — github.com/aiming-lab/SkillRL
   Stars: 472 | 语言: Python
   依赖: Azure OpenAI, SentenceTransformers (Qwen3-Embedding-0.6B), GRPO trainer
--------------------------------------------------------------------------------

  关键文件：
    agent_system/memory/skills_only_memory.py  → Skill bank + 双模式检索
    agent_system/memory/skill_updater.py       → LLM驱动的skill演化（失败分析）
    memory_data/search/claude_style_skills_search.json → Skill bank JSON格式
    skill_generation/alfworld.py, search.py, webshop.py → 领域skill生成

  Skill数据结构：
    {
      "general_skills": [
        {"skill_id": "gen_001", "title": "...", "principle": "...", "when_to_apply": "..."}
      ],
      "task_specific_skills": {
        "pick_and_place": [...],
        "clean": [...]
      },
      "common_mistakes": [
        {"description": "...", "how_to_avoid": "..."}
      ]
    }

  检索方式：
    模式1: keyword/template匹配（按task类型，零延迟）
    模式2: semantic embedding（SentenceTransformer + cosine similarity排序）
    动态skill（dyn_前缀）始终包含在检索结果中

  Skill演化：
    SkillUpdater分析失败trajectory
    → LLM生成1-3个新skill
    → 分配唯一dyn_NNN ID防止冲突
    → 验证必要字段（skill_id, title, principle, when_to_apply）

  可复用部分：
    ✅ 双模式检索（template + embedding）
    ✅ Skill JSON schema
    ✅ 动态ID管理
    ✅ 失败驱动的evolution pipeline
    ✅ ALFWorld/WebShop的环境集成代码

2. Voyager — github.com/MineDojo/Voyager
   Stars: 6,756 | 语言: Python + JavaScript
   依赖: ChromaDB, OpenAI, LangChain
--------------------------------------------------------------------------------

  关键文件：
    voyager/agents/skill.py              → SkillManager类（核心）
    voyager/voyager.py                   → 主agent loop
    skill_library/trial1/skill/code/*.js → 存储的skill代码文件
    skill_library/trial1/skill/description/*.txt → Skill描述

  存储架构（三层同步）：
    层1: 内存dict {program_name: {"code": ..., "description": ...}}
    层2: 文件系统 code/*.js + description/*.txt + skills.json索引
    层3: 向量库 ChromaDB + OpenAI embeddings at skill/vectordb/

  检索：
    ChromaDB的similarity_search_with_score(query, k=k)
    用OpenAI embeddings编码task描述
    返回top-k skill代码

  添加skill：
    LLM从代码生成description
    → 处理重复：同名skill自动版本化（V2, V3...）
    → 同步更新三层存储

  可复用部分：
    ✅ SkillManager的add/retrieve API设计
    ✅ ChromaDB集成模式（正式部署时参考）
    ✅ 内存dict + JSON dump（实验阶段参考）
    ✅ 版本化命名处理重复

3. AutoSkill — github.com/ECNU-ICALK/AutoSkill
   Stars: 175 | 语言: Python
   依赖: LLM APIs, YAML configs, BM25
--------------------------------------------------------------------------------

  关键文件：
    AutoSkill4Doc/models.py              → 数据模型(SkillDraft, SkillSpec, SkillLifecycle)
    AutoSkill4Doc/store/registry.py      → 文件系统存储(DocumentRegistry)
    AutoSkill4Doc/store/retrieval.py     → 混合检索(BM25 + vector + boost)
    AutoSkill4Doc/store/versioning.py    → 语义版本管理
    AutoSkill4Doc/stages/merge.py        → Merge实现
    AutoSkill4Doc/stages/extractor.py    → Skill提取
    AutoSkill4Doc/stages/compiler.py     → Skill编译
    SkillBank/                           → 预构建的SKILL.md文件集合

  Lifecycle状态机：
    candidate → draft → evaluating → active → watchlist → deprecated → retired
    每次状态转换记录为SkillLifecycle事件

  检索（最精细的实现）：
    score = 0.1 × BM25词汇匹配
          + 0.9 × vector dot-product相似度
          + 0.15 (if 匹配asset type)
          + 0.15 (if 匹配node ID)
    最低token overlap门槛过滤

  存储：
    文件系统JSON registry
    URL编码的skill ID
    manifest索引
    provenance追踪 + change logs

  可复用部分：
    ✅ Lifecycle状态机（扩展为我们的Active→Archive→Forgotten）
    ✅ 混合检索（BM25 + vector + domain boost）
    ✅ SkillSpec数据模型 + 语义版本管理
    ✅ Merge实现（stages/merge.py）
    ✅ 文件系统registry模式

4. ExpeL — github.com/LeapLabTHU/ExpeL
   Stars: 208 | 语言: Python
   依赖: OpenAI, LangChain, FAISS
--------------------------------------------------------------------------------

  关键文件：
    agent/expel.py          → ExpeL agent（经验管理 + insight检索）
    memory/episode.py       → Trajectory类（切分后的经验）
    insight_extraction.py   → 规则提取pipeline(ADD/EDIT/REMOVE)
    agent/react.py          → 基础ReAct agent
    agent/reflect.py        → 反思agent

  经验池（双池设计）：
    succeeded_trial_history: Dict[str, Trajectory]  → 成功轨迹
    failed_trial_history: Dict[str, Trajectory]     → 失败轨迹
    每个Trajectory: observations, actions, thoughts的并行列表 + embedding key

  Insight/规则管理：
    LLM通过对比成功vs失败轨迹提取规则
    四种操作：
      ADD    → 新增规则
      EDIT   → 修改已有规则
      REMOVE → 删除错误规则
      AGREE  → 已有规则的计数器+1
    规则按AGREE计数器排序（频率越高越靠前）

  检索：
    多策略轮换：task相似度、thought匹配、action/step匹配
    按cosine distance, trajectory长度, thought embeddings重排序

  可复用部分：
    ✅ 成功/失败双池（对应我们的成功提取steps + 失败提取warnings）
    ✅ Insight提取的ADD/EDIT/REMOVE/AGREE + 计数器（对应confidence）
    ✅ 对比成功vs失败的critique模式
    ✅ Trajectory数据结构

  和我们框架的对应：
    ExpeL的insight ≈ 我们的skill
    ExpeL的ADD ≈ 我们的Acquisition
    ExpeL的REMOVE ≈ 我们的Forgetting
    ExpeL的AGREE计数器 ≈ 我们的confidence

    memengine/function/Judge.py          → 判断模块

    memengine/function/Summarizer.py     → 摘要模块
    memengine/function/Trigger.py        → 触发模块
    memengine/function/Truncation.py     → 截断模块

   Stars: 925 | Zettelkasten式记忆
--------------------------------------------------------------------------------

  关键文件：
    agentic_memory/memory_system.py → AgenticMemorySystem类

  MemoryNote结构：
    UUID id, content, keywords, context, tags,
    links(关联note IDs), creation_timestamp,
    last_access_time, retrieval_count, evolution_history

  Zettelkasten实现：
    add_note() → 创建note → 找最多5个相关memory → LLM决定演化动作

  检索: search_agentic() → ChromaDB检索 → 沿links扩展到邻居notes
