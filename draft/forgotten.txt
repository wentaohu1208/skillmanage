================================================================================
                          Forgotten Area 详解
================================================================================

定位

  Skill生命周期的终点站。

  唯一入口: Archive降级（连续max_archive_inactive轮无人召回）
  不参与任何检索
  唯一用途: 新skill进Active前做去重检查，防止重复学习


================================================================================
一、数据结构
================================================================================

  @dataclass
  class ForgottenSkill:
      skill_id: str                     # 唯一标识
      name: str                         # 技能名称
      summary: str                      # 一句话摘要（来自Archive的压缩版）
      forgotten_at: int                 # 遗忘时的round

  极度精简。
  没有完整描述，没有steps，没有warnings，没有meta，没有完整版本备份。
  只留一个"我曾经会过这个"的痕迹。


================================================================================
二、和Archive的区别
================================================================================

                Archive                     Forgotten
  ──────────────────────────────────────────────────────────
  存储内容      压缩版 + 完整版备份          只有名字和一句话摘要
  参与检索      Active检索不到时搜索         不参与任何检索
  能否恢复      能（恢复完整版）             不能
  占用空间      中等                         极小
  用途          等待被召回                   防止重复学习


================================================================================
三、进入Forgotten
================================================================================

  Archive中Skill S连续inactive ≥ max_archive_inactive轮
      │
      ▼
  从Archive中移除S
      │
      ▼
  丢弃:
    完整版本（original_skill_full）
    压缩版详情
    所有meta
      │
      ▼
  只保留:
    skill_id
    name
    summary（一句话，来自Archive的压缩版）
    forgotten_at = current_round
      │
      ▼
  存入Forgotten Pool


================================================================================
四、唯一用途：去重检查
================================================================================

  Acquisition提取了一个新skill，准备进Active
      │
      ▼
  和Forgotten Pool做语义匹配
  sim(new_skill.description, forgotten.summary)
      │
      ├─ sim < τ (如0.8) → 全新skill → 正常进Active
      │
      └─ sim ≥ τ → 以前学过这个skill
           │
           ▼
      forgotten_at距今多久？
           │
      ┌────┴────┐
      │         │
    < D轮     ≥ D轮
      │         │
      ▼         ▼
    跳过       允许进Active
   (最近才     (淘汰很久了，
    淘汰，      任务分布可能
    大概率      已经变了，
    还是没用)   值得重新试试)


================================================================================
五、为什么不直接删除，要留Forgotten
================================================================================

  不留Forgotten的情况：

    Round 230: heat_object被彻底删除，什么都不留
    Round 260: Agent又从trajectory中提取了"加热物品"
               → 没有去重检查
               → 走完整个Acquisition流程
                 （积累M条记录，Alignment，Formalization）
               → 进Active
               → 用了几轮发现还是没用
               → 又降级到Archive → 又到Forgotten
               → 循环浪费

  留了Forgotten的情况：

    Round 260: 检查Forgotten → 发现以前学过 → 跳过
               → 省了一整轮Acquisition + Active + Archive的开销

  Forgotten本质上是一个"黑名单"，防止agent反复学习同一个没用的skill。


================================================================================
六、具体例子
================================================================================

  Round 142: Skill "heat_object" 从Active降级到Archive
  Round 230: 在Archive中50轮无人召回 → 降级到Forgotten
             Forgotten Pool中记录:
               {
                 skill_id: "sk_017",
                 name: "heat_object",
                 summary: "用microwave加热物品",
                 forgotten_at: 230
               }

  ─────────────────────────────────────────────────

  Round 260: Agent从trajectory中又提取了 "用microwave加热食物"
             → 进Active前查Forgotten Pool
             → 匹配到 "heat_object" (sim=0.92 > τ=0.8)
             → forgotten_at=230, 距今30轮 < D(100轮)
             → 跳过，不进Active
             → 30轮前才淘汰的，当前任务分布下大概率还是没用

  ─────────────────────────────────────────────────

  Round 400: Agent又提取了类似skill
             → 查Forgotten Pool
             → 匹配到 "heat_object" (sim=0.90 > τ=0.8)
             → forgotten_at=230, 距今170轮 ≥ D(100轮)
             → 允许进Active
             → 淘汰了170轮了，任务分布可能早就变了，值得重新试


================================================================================
七、在整个Lifecycle中的位置
================================================================================

  Acquisition提取新skill
      │
      ▼
  查Forgotten去重 ←─────────────────────────────────┐
      │                                               │
      ├─ 匹配到且最近淘汰 → 跳过                      │
      ├─ 匹配到但淘汰很久 → 允许进Active               │
      └─ 没匹配到 → 正常进Active                      │
                                                       │
  Active                                               │
      │                                               │
   importance低                                        │
      │                                               │
      ▼                                               │
  Archive                                              │
      │                                               │
   长期无人召回                                         │
      │                                               │
      ▼                                               │
  Forgotten ────────────────────────────────────────────┘
  (去重检查)


================================================================================
八、调API的地方
================================================================================

  ┌──────────────────────────┬──────────┬────────────────────────┐
  │ 操作                      │ 调API？  │ 说明                    │
  ├──────────────────────────┼──────────┼────────────────────────┤
  │ 从Archive降级到Forgotten  │ ❌       │ 只是丢弃数据保留摘要    │
  │ 去重匹配                  │ ❌       │ embedding cosine，本地  │
  │ 时间衰减判断              │ ❌       │ 数值比较                │
  └──────────────────────────┴──────────┴────────────────────────┘

  Forgotten的API开销为零。所有操作都是本地计算。


================================================================================
九、超参数
================================================================================

  参数    含义                                建议值
  ─────────────────────────────────────────────────────
  τ       去重匹配的相似度阈值                 0.8
  D       时间衰减轮数（超过D轮允许重新学习）   100
